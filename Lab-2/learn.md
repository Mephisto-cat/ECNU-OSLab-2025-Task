# Lab-2 学习笔记

## 1. 物理页分配器：空闲链表

**问题**：128MB 物理内存，内核如何追踪哪些页空闲、哪些已分配？

**方案**：带头结点的单链表。每个空闲页的前 8 字节被复用为 `struct run *next` 指针——不需要额外内存来维护链表。

三个接口都是 O(1)：
- `kfree` 是头插法（拿到一个物理地址，强制转成 `struct run *`，挂到链表头部）
- `kalloc` 从头部摘下，**清零后再返回**（防止上一任用户的数据泄露）
- `kinit` 遍历 `[PGROUNDUP(end), PHYSTOP)`，把所有页依次 `kfree` 进链表

并发安全靠 `spinlock`——Lab-1 写的锁在这里直接复用。

## 2. void * 通用指针

`void *` 是 C 的"泛型"——指向任意类型的指针。分配器不知道调用者会把一页用来存什么（页表？字符串？结构体？），所以返回 `void *`，由调用者自己强制转换。任何指针可以隐式转为 `void *`，反过来必须显式强转。

## 3. 页表就是翻译字典

程序用虚拟地址，物理内存只有一块。页表负责把"程序看到的假地址"翻译成"内存条上的真地址"。RISC-V Sv39 用三级页表（每级 512 项，每项 8 字节），每级用虚拟地址的 9 位做索引：

```
va[38:30] → 查 L2 页表 → 得到 L1 页表地址
va[29:21] → 查 L1 页表 → 得到 L0 页表地址  
va[20:12] → 查 L0 页表 → 得到物理页地址
va[11:0]  → 页内偏移
```

"三级"就是查三次表。分级的好处：大部分地址空间是空的，对应的中间级页表根本不需要分配，省内存。

## 4. 内核页表采用对等映射

内核页表中 **虚拟地址 = 物理地址**（identity mapping）。好处：开启分页后内核代码完全不用改——`0x80000000` 翻译完还是 `0x80000000`。但内核页表也映射了所有 RAM（128MB）和 UART MMIO，所以内核能访问任何内存和外设。

## 5. PTE 内部布局

一个 PTE（页表项）是 64 位：高 44 位存物理页号（PPN），低 8 位是标志位（V=有效, R=读, W=写, X=执行, U=用户态）。`PA2PTE` / `PTE2PA` 负责在"物理地址"和"PTE 内容"之间做位位移转换——PPN 在物理地址的 bit[55:12]，在 PTE 的 bit[53:10]。

## 6. satp 寄存器

satp 是 S-mode 的页表基址寄存器：bit[63:60] 设成 8 表示 Sv39 模式，低 44 位填根页表的 PPN。`kvminithart()` 最后写入 satp 的瞬间，CPU 的 MMU 就开始工作了。每个 hart 有自己的一套寄存器，所以 `kvminithart()` 必须每个 hart 各调一次。

## 7. 用测试证明实验完成

- 耗尽全部内存确认 kalloc 返回 NULL
